SpringBoot 中使用 AOP（五）

发表于 2017-07-22 | 分类于 后端开发

切面类无效

一般情况下，项目中会存在一个 Spring 的主配置文件 applicationContext.xml 和若干辅配置文件，如配置 SpringMVC 的 spring-servlet.xml。我们可以在任意一个 Spring 的配置文件中去配置 Spring 的包扫描。配置包扫描的配置项和配置 AOP 自动扫描的配置项必须在同一个配置文件中，而且包扫描配置项不要在不同文件中重复配置。如: <aop:aspectj-autoproxy /> 声明自动为 Spring 容器中那些配置了 @AspectJ 注解的 POJO 创建代理，织入切面。<context:component-scan base-package="pers.vincent.controller"></context:component-scan> 则是自动扫描包。这两个一定要在同一个配置文件中。如果是通过 @Component 注解实现，那么可以忽视，以上说的是通过 xml 配置实现。

切入点方法执行多次

1.先检查配置文件中是否有重复的包扫描？譬如我在不同的 Spring 配置文件中都声明了 <context:component-scan base-package="pers.vincent.controller"></context:component-scan> 配置项。2. 被切入点代理的类是否存在继承关系，譬如我想代理 pers.vincent.controller 包下的全部方法，而该包下的全部类都继承与 pers.vincent.controller.BaseController，这种情况就有可能导致切入点方法执行两次。解决方法: 可以通过声明一个 @Pointcut(value = "pers.vincent.controller.BaseController.*.(..)") 切入点。然后通过切入点组合（详情见SpringBoot中使用AOP（三）），将该类排除出去。

@AfterThrowing 不起作用

1.被织入的方法必须抛出异常，而不是被 try catch 吃掉，才会进入到 @AfterThrowing 注解的方法中去。2.假如目标方法的通知类型为 @Around(环绕通知) 那么通知方法，即被 @Around 也必须将异常抛出，@AfterThrowing 才可以正常进入。

唯独 Controller 类无法通知

<aop:aspectj-autoproxy /> 有一个 proxy-target-class 属性，默认为 false，表示使用 jdk 动态代理织入增强，当配为 <aop:aspectj-autoproxy poxy-target-class="true"/> 时，表示使用 CGLib 动态代理技术织入增强。不过即使 proxy-target-class 设置为 false，如果目标类没有声明接口，则 Spring 将自动使用 CGLib动态代理。
