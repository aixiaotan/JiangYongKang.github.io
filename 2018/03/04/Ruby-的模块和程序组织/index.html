Ruby 的模块和程序组织

发表于 2018-03-04 | 分类于 开发者基础

和类一样，模块是一组方法和常量的集合。和类不同，模块没有实例，取而代之的是可以将特殊模块的功能添加到一个类或者指定对象之中。Class 类是 Module 类的一个子类。因此每一个类对象也是一个模块对象。

创建模块

如何创建一个模块？与创建一个类相似，只是将 class 关键字替换为 module 即可。

通常情况下，编写完一个类之后，我们可以创建类的实例，这些实例可以执行类的实例方法。模块是没有实例的，模块采用混合 mix in 到类的方式。即可以使用 include 方法，也可以使用 perpend 方法。

ModuleTester 类混合了模块 FirstModule，那么它的实例对象就可以调用 FirstModule 中的方法。一旦需要为类的实例增加大量的附加行为，并且不想将他们存储在本类或者超类中，那么模块将是一个很好的选择。

require/load 和 include

当我们使用 require/load 的时候，通常会把需要加载的项目放在引号中，而 include/perpend 确是提供一个常量。这是因为 require/load 通常是从磁盘中加载文件，而 include/perpend 是从内存中加载模块。注意：类名通常是一个名词而模块名通常是一个形容词。

模块、方法、类、常量的查找顺序

每当调用方法，预期的结局就是在对象的实例方法、对象超类的实例方法(依次往上追溯，到达 Object 甚至 BasicObject 类中)，或者在混合到类的模块中，执行与消息同名的方法。如果在本类、超类、模块、超类的模块中都未找到想要执行的方法，则会发生一个错误，错误条件由一个 method_missing 触发，它作为最后没有匹配到任何方法的处理手段。
定义的类会默认的继承 Object，而 Object 的超类是 BasicObject，BasicObject 混合了 Kernel 模块。

同名方法的多次定义

在类中多次定义同一个方法，不会抛出异常，程序能够正常运行，但是第二次定义的方法会覆盖第一次定义的方法。这在模块中依然有效。

方法的查找路径上面已经提到了，假如对象的查找路径中包含多个同名方法，那么会执行首次找到的方法。好比类中混合了一个模块，假如类中和模块中存在同名方法，实际调用时会执行类中的那个方法，而不是模块中的，因为 ruby 会先在类中查找方法，其次才是模块中。

那么包含了多个模块，多个模块中定义了同名方法，它该执行哪一个呢？答案是会执行后 include 进来的模块中的方法。可以通过 FirstClass.included_modules 来查看混合了哪些模块，会发现后混合进来的模块在数组的前面，也就是说会先在这个模块中查找方法。

因此可以总结出 ruby 方法查找的顺序:

  • 被前置在它所属类中的模块，按照与前置相反的顺序
  • 它的类中
  • 被包含在它所属类中的模块，按照与包含相反的顺序
  • 前置到它超类中的模块
  • 它所属的超类中
  • 包含在它所属超类中的模块
