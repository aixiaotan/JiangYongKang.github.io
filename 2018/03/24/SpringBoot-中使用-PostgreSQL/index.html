SpringBoot 中使用 PostgreSQL

发表于 2018-03-24 | 分类于 后端开发 |

与 MySQL、Oracle 相比，PostgreSQL 算是比较冷门的一个数据库。在网上查了很多关于如何集成 SpringBoot 和 PostgreSQL 的资料，要么版本太低，要么配置有误。为此整理了一篇关于如何在 SpringBoot 中使用 PostgreSQL 的文章。

版本的选择

首先 SpringBoot 的版本可以任意选择，本文使用的是 2.0.0.BUILD-SNAPSHOT，PostgreSQL 的版本是 9.6.4。

添加依赖

如何选择 PostgreSQL 的数据库驱动版本？官网 很详细的介绍了驱动文件对 Java 版本的支持。我用的是 Java 1.8 这里选择的是 42.2.2 的驱动。

配置项目属性

成功导入所必须的依赖之后，我们就可以配置项目的属性了，先看看 PostgreSQL 数据库连接的写法。

创建关系映射对象

创建 Repository 接口

JPA 支持两种表达查询的方法来检索实体和来自数据库的其他持久化数据，继承 CrudRepository 接口可以获得基本的增删改查的方法。如果不够用可以通过 JPQL 语法定义简单的查询，复杂查询需要通过 @Query 注解自己写查询。
