nginx 多端口映射

发表于 2018-06-07 | 分类于 后端开发 |

我们通过域名访问服务器资源的时候，会先将域名解析为IP，然后访问80端口。比如在浏览器输入 https://www.vincent.com 实际上访问的是 xxx.xxx.xxx.xxx:80。我们总不能每次在输入域名之后再敲一个端口号吧？于是我们就可以通过 nginx 来反向代理本地的 web server 端口。

比如我现在有三个二级域名，分别是 uat1.vincent.com uat2.vincent.com uat3.vincent.com，分别将他们转发至本机的 3001，3002，3003 端口。

配置完成后重启 nginx

在浏览器输入 uat1.vincent.com uat2.vincent.com uat3.vincent.com ，不出意外就能分别访问各自的 web server 了。
