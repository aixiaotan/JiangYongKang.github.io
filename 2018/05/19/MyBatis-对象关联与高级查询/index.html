MyBatis 对象关联与高级查询

发表于 2018-05-19 | 分类于 后端开发 |

MyBatis 的真正强大在于它的映射语句，这是它的魔力所在。我们通过映射语句，可以减少 90% 的 JDBC 样板代码，但同时也会产生大量的 XML 映射文件，这是无法避免的。想要拥有对于 SQL 语句较高的掌控力，势必会用到 MyBatis 映射文件，即便现在已经有很多注解。本文主要讲解如何去合理的使用 XML 提高它的可读性、重用性。

基本的查询

查询语句开发过程中最常用但也最容易出错的语句，MyBatis 赋予了我们自己编写 SQL 语句的权利，可以通过 @Select 注解或者 <select></select> 声明一条查询语句。

灵活的 resultMap

上面的例子在简单的一对多查询上比较方便，往往真实的业务场景下对象的关联层级是比较深的，我们不可能一直的去 resultMap -> resultMap -> resultMap 无节制的嵌套下去，也不能 LEFT JOIN -> LEFT JOIN -> LEFT JOIN 无节制的嵌套下去，这样下去 resultMap 会堆积的无比复杂，看起来会让你望而生畏，而且性能也会有一定损失。
所幸的是 resultMap 提供了引用外部查询语句的机智，可以让你脱离 resultMap 的嵌套地狱。

看起来似乎没有太多的变化，但是这种通过执行另外一个 SQL 映射语句来返回预期的复杂类型相对于使用嵌套结果映射来处理重复的联合结果的子集的耦合度要低很多。而且我们多了一条可以被重用的 SQL 语句 selectBookForAuthor。

总结

MyBatis 在加载关联对象这方面会有三种不同的方式：

  • 直接内嵌查询结果映射
  • 引用外部另一个 resultMap 处理关联属性
  • 通过执行另外一个 SQL 映射语句来返回预期的复杂类型
