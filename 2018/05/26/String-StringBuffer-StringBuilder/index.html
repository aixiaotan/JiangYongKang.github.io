String StringBuffer StringBuilder

发表于 2018-05-26 | 分类于 开发者基础 |

在编写 Java 代码的过程中有时要频繁地对字符串进行拼接，如果直接用 + 拼接的话会建立很多的 String 型对象，严重的话会对服务器资源和性能造成不小的影响。而使用 StringBuilder 和 StringBuffer 能解决以上问题。

String

众所周知 String 作为不可修改的对象，即每次更改 String 都会产生一个新的字符串对象。自 Java 1.5 以来，首先是新增了 StringBuilder 类，其次是对类似 s = s + "x" + "y" + "z" 这种加号拼接字符串的方式做了优化。

StringBuffer

StringBuffer 经常用来拼接字符串，它在很早的时候就有的，但是它是线程安全的，所以效率比 String 要高一些，至少它不会每次拼接都创建新的对象，它的 append() 方法，返回的都是 this。

StringBuilder

StringBuilder 是 Java 1.5 之后才推出的 API，它可以更加高效的去拼接字符串，但是它是非线程安全的。也就意味着在多线程环境下去拼接，可能会出现问题。这点需要注意。
