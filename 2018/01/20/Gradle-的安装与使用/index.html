Gradle 的安装与使用

发表于 2018-01-20 | 分类于 后端开发 |

Gradle 是一个基于 Apache Ant 和 Apache Maven 概念的项目自动化构建工具。它使用一种基于 Groovy 的特定领域语言(DSL) 来声明项目设置，抛弃了基于 XML 的各种繁琐配置。面向 Java 应用为主。当前其支持的语言限于 Java、Groovy、Kotlin 和 Scala，计划未来将支持更多的语言。

如何安装

在 Mac OS 上推荐使用 brew 安装。通过 brew info gradle 命令可以查看详细信息：

可以看到 Gradle 是依赖与 JDK 的，所以在安装 Gradle 之前必须配置好自己的 Java 开发环境，然后可以通过 brew install gradle 进行安装。

安装好的文件在 /usr/local/Cellar/gradle 这个位置，我们需要配置它的环境变量。

配置完环境变量之后，可以输入 gradle -v 来验证安装是否成功。

构建脚本

Gradle 的构建脚本有三种，分别是 build.gradle gradle.setting gradle.properties。

  • build.gradle 是项目的构建文件，声明了项目在编译、测试、运行阶段的依赖，等同于 Maven 中的 pom.xml
  • gradle.setting 是用于多项目构建时
  • gradle.properties 声明项目的一些属性，比如项目的版本等等。

依赖冲突

一般的冲突 Gradle 会帮我们自动的解决，并不需要我们干预。它会在默认选择冲突的库文件中取高版本的那个。比如：spring.jar 依赖于 slf4j-1.0.jar，而 hibernate.jar 依赖于 slf4j-1.5.jar。那么这两个不同的版本就产生了冲突，所幸的是，这种冲突并不需要我们自己解决，Gradle 会选取高版本的那一个作为传递性依赖，即 slf4j-1.5.jar。
