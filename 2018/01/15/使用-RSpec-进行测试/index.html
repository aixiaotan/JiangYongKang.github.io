使用 RSpec 进行测试

发表于 2018-01-15 | 分类于 后端开发

RSpec 是一套 Ruby 的测试 DSL(Domain-specific language) 框架，它的程式比 Test::Unit 更好读，写的人更容易描述测试目的，可以说是一种可执行的规格文件。也 非常多的 Ruby on Rails 专案采用 RSpec 作为测试框架。它又称为一种 BDD(Behavior-driven development) 测试框架，相较于 TDD 用 test 思维，测试程式的结果。 BDD 强调的是用 spec 思维，描述程式应该有什么行为。

在项目中初始化 RSpec

如果是 Ruby 项目，执行 rspec --init。这个命令会帮我们自动生成 spec 文件夹和 spec_helper.rb 文件。我们要编写的测试代码都位于 spec/ 文件夹下并且以 _spec 作为文件名的后缀。比如测试 Book 这个类就应该建立 spec/book_spec.rb 文件。

在 Rails 项目中使用 Rspec。找到 group :development, :test do 这个分组，添加依赖。

基本的测试

  • describe 用来描述测试的行为，一般用对应测试类名或者模块名，没有的情况下可以用字符串。
  • it 对应着一个测试 case，在一个 describe 中可以声明多个 it。
  • expect 用来断言实际的数值是否符合预期，expect(左值).to eq(右值)

使用 before 和 let

  • before :each 在每个 it 执行前都会被调用一次。before :all 只会执行一次
  • let(:first_account) 是懒加载机制，只有在被调用的时候才会被初始化。

使用 context 测试不同场景

  • context 只是 describe 的别名，使用方式完全一致。
  • context 只是为了让代码的可读性更高。
