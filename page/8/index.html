SpringBoot 中使用 AOP Part-5

发表于 2017-07-22 | 分类于 后端开发

切面类无效

一般情况下，项目中会存在一个 Spring 的主配置文件 applicationContext.xml 和若干辅配置文件，如配置 SpringMVC 的 spring-servlet.xml。我们可以在任意一个 Spring 的配置文件中去配置 Spring 的包扫描。配置包扫描的配置项和配置 AOP 自动扫描的配置项必须在同一个配置文件中，而且包扫描配置项不要在不同文件中重复配置。如: <aop:aspectj-autoproxy /> 声明自动为 Spring 容器中那些配置了 @AspectJ 注解的 POJO 创建代理，织入切面。<context:component-scan base-package="pers.vincent.controller"></context:component-scan> 则是自动扫描包。这两个一定要在同一个配置文件中。如果是通过 @Component 注解实现，那么可以忽视，以上说的是通过 xml 配置实现。
